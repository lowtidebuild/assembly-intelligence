## Problem Statement

게임산업 GR 담당자가 브리핑봇을 썼을 때 "게임산업법" 관련 발의안만 잡히고 "전자상거래법", "표시·광고의 공정화법", "이스포츠 진흥법", "정보통신망법" 등 게임사에 직접 영향을 주는 교차 법률 발의안은 누락됨. 현재 `src/lib/industry-presets.ts`의 pre-filter는 산업별 `keywords: string[]` 배열 하나만 보므로, 산업이 복수 법률을 가로지르는 경우를 구조적으로 처리하지 못함.

실무자(게임사 GR 담당자 1명에게 초기 확인됨)는 설정 UI에서 "이 산업이 관련된 법률들"을 직접 체크박스로 골라 watchlist를 넓히고 싶어함.

## Demand Evidence

- 실사용자(assembly-intelligence 내부 사용자) 명시적 피드백: "게임산업법만 뜨는데 전자상거래법/표시광고법/이스포츠법/정보통신망법도 넣고 싶다"
- 교차 법률 니즈는 게임에 국한되지 않음. 바이오(약사법 + 개인정보보호법 + 생명윤리법), 핀테크(전자금융법 + 신용정보법 + 개인정보보호법), AI(AI기본법 + 저작권법 + 개인정보보호법) 모두 동일한 구조적 요구.

## Status Quo

`src/lib/industry-presets.ts`에 7개 프리셋(game, cybersecurity, bio, fintech, semiconductor, commerce, ai) 각각이 자기 `keywords`와 `excludeKeywords`를 가짐. 일부 법률 키워드는 여러 프리셋에 중복 존재하지만 구조적 재사용은 없음.

- commerce 프리셋: "전자상거래", "전자상거래법", "표시·광고", ...
- cybersecurity 프리셋: "정보통신망", "정보통신망법", "개인정보 보호", ...
- ai 프리셋: "저작권법"(llm_context에만), ...

game 프리셋이 이들 키워드를 쓰려면 오늘은 game 프리셋을 직접 편집(`game-v1.2`)하는 수밖에 없음. 다른 산업 담당자도 마찬가지. 확장성 0.

## Agreed Premises

1. **실무자 페인 실재:** 게임사 GR 담당자는 게임산업법 외 교차 법률을 묶어 추적하고 싶음. (assembly-intelligence 내부 사용자 1명 확인)
2. **확장성 요구:** 게임뿐 아니라 다른 산업도 교차 법률이 필요. 플랫폼 기능이어야 하며 게임 전용 패치가 아님.
3. **DRY 원칙:** commerce/cybersecurity 프리셋에 이미 잘 정돈된 법률 키워드 블록을 재사용. 새로 쓰지 않음.
4. **충돌 최소:** 현재 commerce의 excludeKeywords가 비어 있어 게임에 커머스 법률을 믹스인해도 배제 충돌 없음. (향후 excludeKeywords 채워질 경우 병합 정책 필요 — Open Question 참조.)

## Chosen Approach: Law Mixin Registry (A)

별도 `src/lib/law-mixins.ts` 레지스트리 신설. 프리셋은 무손상.

### 데이터 모델

```ts
// src/lib/law-mixins.ts
export interface LawMixin {
  slug: string;                // e.g. "ecommerce-act"
  name: string;                // "전자상거래법"
  formalName: string;          // "전자상거래 등에서의 소비자보호에 관한 법률"
  keywords: string[];          // pre-filter terms
  excludeKeywords: string[];   // false-positive guards
  regulators: string[];        // "공정거래위원회" — UI 설명용, 선택적
  version: string;             // "ecommerce-act-v1.0"
}

export const LAW_MIXINS: Record<string, LawMixin> = {
  "ecommerce-act": { /* 전자상거래법 */ },
  "fair-labeling-act": { /* 표시·광고의 공정화법 */ },
  "info-comm-network-act": { /* 정보통신망법 */ },
  "esports-promotion-act": { /* 이스포츠 진흥법 */ },
  "pipa": { /* 개인정보보호법 */ },
  "copyright-act": { /* 저작권법 */ },
  // ... 초기 6-8개. 나머지는 수요에 따라 추가.
};
```

### DB 스키마

`industry_profile` 테이블에 컬럼 추가:

```ts
selectedLawMixins: text('selected_law_mixins', { mode: 'json' }).$type<string[]>().default([]),
```

(SQLite 호환 JSON 배열. `string[]`에 law mixin slugs 저장.)

### Pre-filter 병합 로직

`src/services/sync.ts`의 키워드 매칭 단계에서:

```
effectiveKeywords = profile.keywords
                  ∪ flatMap(profile.selectedLawMixins → LAW_MIXINS[slug].keywords)

effectiveExcludes = profile.excludeKeywords
                  ∪ flatMap(profile.selectedLawMixins → LAW_MIXINS[slug].excludeKeywords)
```

### UX — Setup Wizard Step 1

산업 프리셋 선택 카드 아래에 "**관련 법률 추가**" 섹션 신설.
게임 선택 후: 기본 체크됨 = 없음. 체크박스 목록:
- ☐ 전자상거래법
- ☐ 표시·광고의 공정화법
- ☐ 정보통신망법
- ☐ 이스포츠 진흥법
- ☐ 개인정보보호법
- ☐ 저작권법
- ...

각 체크박스에 tooltip: "이 법률을 추가하면 관련 키워드 N개가 watchlist에 더해집니다. (예: '전자상거래', '오픈마켓', '다크패턴' ...)"

기존 프리셋(game-v1.1)의 키워드는 그대로 유지. 사용자는 "게임 + 전자상거래법 + 정보통신망법"처럼 조합.

### 마이그레이션

- 신규 프로필: 빈 `selectedLawMixins: []`.
- 기존 프로필: 컬럼 추가, 기본값 빈 배열. 기존 동작 변화 없음.
- 사용자는 `/settings` 또는 `/setup` 재진입으로 업그레이드.

## Target User & Scope

**Primary user:** GR/PA 담당자(게임·바이오·핀테크·AI 등 복수 산업). 현재 assembly-intelligence 내부 사용자 기준 설계, 외부 확장 가능.

**In scope:**
- `law-mixins.ts` 초기 6-8개 믹스인 정의 (전자상거래법, 표시광고법, 정보통신망법, 이스포츠진흥법, 개인정보보호법, 저작권법 최소)

- 테스트: 병합 로직 단위 테스트, setup-wizard 통합 테스트

**NOT in scope (defer):**

- Law mixin별 LLM context 주입 — Gemini 점수 프롬프트 구조 변경 필요, 별도 plan

## Open Questions

1. **excludeKeywords 충돌 정책:** 산업 프리셋의 excludeKeywords와 mixin의 excludeKeywords가 겹치면 union(현 설계). 상충하는 경우(예: 산업은 "X" 포함, mixin은 "X" 배제)는 현재 없지만 향후 발생시 프리셋 우선 규칙 필요.
2. **Mixin versioning:** 프리셋은 `presetVersion` 있음. Mixin도 동일한 drift 감지 필요. 간단히 `version: "ecommerce-act-v1.0"` 달고, 사용자 프로필에 선택 시점 스냅샷 저장할지 결정 필요(지금은 미보존, 항상 최신 적용).
3. **초기 6-8개 법률 목록:** 위 예시 외 추가 필요? (예: 콘텐츠산업진흥법, 청소년보호법, 약사법, 전자금융거래법 ...) → Plan 단계에서 확정.

## Assignment

Plan 작성 전에 게임사 GR 담당자 1명에게 이 6-8개 법률 목록 보여주고 "이 중 빠진 거, 필요 없는 거" 피드백 받기. 10분이면 됨. 이 피드백이 초기 `LAW_MIXINS` 엔트리 목록을 결정.
